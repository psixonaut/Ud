# CRM Автосалона - Система управления автомобильным дилерским центром

## 📋 Описание проекта

Курсовой проект по предмету "Управление данными" - полнофункциональная система управления взаимоотношениями с клиентами (CRM) для автомобильного салона. Система обеспечивает управление инвентарем автомобилей, обработку заказов, оформление продаж, организацию тест-драйвов и управление персоналом салона.

**Статус**: Курсовой проект  
**Язык**: Python (Django)  
**База данных**: PostgreSQL

---

## 🎯 Основные возможности

### 1. **Управление персоналом**
- Добавление и удаление сотрудников
- Три категории должностей: Менеджер, Продавец-консультант, Специалист по закупкам
- Управление статусом занятости (трудоустройство/увольнение)
- Хранение персональных данных (ФИО, паспорт, водительское удостоверение, телефон)

### 2. **Управление клиентами**
- Регистрация новых клиентов
- Хранение информации о клиентах (паспортные данные, контакты, возраст)
- Интеграция с системой тест-драйвов и продаж

### 3. **Управление инвентарем автомобилей**
- Каталог всех автомобилей с детальной информацией (марка, модель, VIN, тип кузова)
- Отслеживание технических характеристик (тип трансмиссии, тип привода, объем двигателя)
- Система статусов автомобилей:
  - **В пути** - в процессе доставки
  - **Прибыл** - поступил на склад
  - **В продаже** - доступен для продажи
  - **Для тест-драйвов** - зарезервирован для тестирования
  - **Продан** - реализован

### 4. **Управление закупками**
- Оформление заказов у поставщиков (только Специалист по закупкам)
- Отслеживание статуса заказов:
  - **В пути** - в пути от поставщика
  - **Прибыл** - получен в салон
  - **Оформлен** - оформлен в системе
- Приемка автомобилей с применением скидок

### 5. **Система продаж**
- Оформление продаж клиентам (только Менеджеры и Продавцы-консультанты)
- Управление скидками (максимум 50%)
- Отслеживание итоговой суммы сделки
- Автоматический расчет цены со скидкой
- Переназначение автомобилей между заказами и продажами

### 6. **Организация тест-драйвов**
- Бронирование автомобилей на тест-драйв (минимум 21 год для клиента)
- Привязка только сотрудников с водительским удостоверением
- Система результатов:
  - **Ожидается** - запланирован
  - **Проведён** - завершен
  - **Успешно** - клиент доволен
  - **Клиент думает** - требуется решение
  - **Отказ** - клиент отказался
  - **Купил** - завершился покупкой
- Автоматическое обновление статуса по дате/времени

### 7. **Аутентификация и авторизация**
- Система входа для сотрудников
- Проверка статуса занятости сотрудника
- Разделение доступа по должностям
- Защита от действий уволенных сотрудников

## 🔒 Бизнес-логика и ограничения

### Требования для операций:

**Закупка автомобилей:**
- Только "Специалист по закупкам" может создавать заказы
- Минимальная цена со скидкой: 100,000 руб.

**Продажа автомобилей:**
- Только "Менеджер" и "Продавец-консультант" могут оформлять продажи
- Только автомобили со статусом "В продаже" можно продать
- Скидка не может превышать 50%
- Минимальная цена на продажу: 100,000 руб.

**Тест-драйвы:**
- Минимальный возраст клиента: 21 год
- Клиент должен иметь водительское удостоверение
- Сотрудник должен иметь водительское удостоверение
- Только "Менеджер" и "Продавец-консультант" могут проводить тест-драйвы
- Автомобиль должен иметь статус "Для тест-драйвов"
- Статус автоматически обновляется после указанной даты/времени

**Требования к сотрудникам:**
- Минимальный возраст: 18 лет
- Все данные должны быть уникальны (номер паспорта, номер ВУ, телефон)

## 🚀 Установка и запуск

### Предварительные требования:
- Python 3.8 или выше
- PostgreSQL
- pip

### Шаги установки:

1. **Клонирование репозитория**
```bash
git clone <repository-url>
cd Ud
```

2. **Создание виртуального окружения**
```bash
python -m venv venv
source venv/bin/activate  # На Windows: venv\Scripts\activate
```

3. **Установка зависимостей**
```bash
pip install -r requirements.txt
```

4. **Конфигурация базы данных**
   - Создать БД PostgreSQL
   - Обновить параметры подключения в `config/settings.py`

5. **Применение миграций**
```bash
python manage.py migrate
```

6. **Создание суперпользователя**
```bash
python manage.py createsuperuser
```

7. **Запуск сервера**
```bash
python manage.py runserver
```

Приложение будет доступно по адресу: `http://localhost:8000/`

## 📝 Примеры функционала

### Создание продажи:
1. Выберите клиента (или создайте нового)
2. Выберите автомобиль со статусом "В продаже"
3. Укажите скидку (если необходимо)
4. Система автоматически рассчитает финальную цену
5. Статус автомобиля изменится на "Продан"

### Бронирование тест-драйва:
1. Выберите автомобиль со статусом "Для тест-драйвов"
2. Выберите клиента (возраст должен быть ≥ 21)
3. Укажите дату и время
4. Система автоматически обновит статус после указанного времени
